# DrawNet — Developer Handover

## What is this project?

DrawNet is a multi-task deep learning model for sketch analysis. Given a sketch image as input it simultaneously:
1. **Classifies the subject** (intent recognition — 30 categories)
2. **Detects geometric drawing deviations** (multi-label — 4 binary classes)

Built with PyTorch. ResNet-50 backbone, two task-specific heads. Trained on QuickDraw + TU-Berlin sketch datasets.

Read [CONTEXT.md](CONTEXT.md) for the full design rationale, architecture diagram, category map, and dataset details.

## Running the Pipeline

Run these four commands in order. Each step is a prerequisite for the next.

### Step 1 — Build annotations
```bash
cd drawnet/
python src/build_annotations.py --show_stats
```
**What it does:** Scans QuickDraw `.npy` files and TU-Berlin image folders. Assigns intent labels (0–29) from category names. Randomly flags exactly 20% of each category as `will_deviate=1`.

**Output:** `data/processed/master_annotations.csv`
- ~64,158 rows (50,000 QuickDraw + 14,158 TU-Berlin)
- Columns: `source, filepath, npy_row_index, intent_label, category, will_deviate`
- ~20% of rows have `will_deviate=1`

**Verify:** The stats output should show ~20% deviated per category and all 30 categories present.

---

### Step 2 — Cache dataset
```bash
python src/cache_dataset.py
```
**What it does:** Reads `master_annotations.csv`. For each row: loads the image (QuickDraw bitmap or TU-Berlin JPEG), applies deviation augmentation if `will_deviate=1` (using `augment.py`), saves as PNG.

**Output:**
- `data/augmented/images/000000.png … N.png` (~64k PNGs)
- `data/augmented/index.csv` — filepath + intent_label + 4 deviation columns

**Time:** ~10–15 minutes on local machine. Supports `--no_resume` to restart, resumes by default if interrupted.

**Verify:** Deviation distribution in the printed summary should show:
- `rotation, closure_failure, spatial_disorganization, size_distortion` each ~10–14% of total
- "Any deviation" ~20% of total

---

### Step 3 — Split dataset
```bash
python src/split_dataset.py --show_stats
```
**What it does:** Stratified 80/20 train/test split on `index.csv`, stratified by `intent_label` so every category appears in both splits proportionally.

**Output:**
- `data/augmented/train.csv` (~51,300 rows)
- `data/augmented/test.csv`  (~12,800 rows)

**Verify:** Both splits should show ~20% deviated rate. Every intent class should appear in both.

---

### Step 4 — Train
```bash
python src/train.py
# or on Colab:
python src/train.py --config configs/config_colab.yaml
```
**What it does:** Reads `train.csv`, splits internally 80/20 into train/val. Trains for 50 epochs with progressive backbone unfreezing. Saves best checkpoint to `outputs/checkpoints/best.pt`.

**Resume from checkpoint:**
```bash
python src/train.py --resume outputs/checkpoints/epoch_020.pt
```

**Backbone unfreezing schedule:**
- Epochs 1–10: backbone frozen, heads only
- Epochs 11–20: unfreeze ResNet layer4
- Epochs 21–50: unfreeze full backbone at LR=1e-5

**Key config settings (config.yaml):**
- `model.pretrained: false` (Windows SSL blocks download — set `true` on Colab)
- `training.batch_size: 32` (RTX 3050 4GB — increase to 64 on Colab)
- `training.num_workers: 2`

**Outputs:**
- `outputs/checkpoints/best.pt` — best validation loss checkpoint
- `outputs/checkpoints/epoch_010.pt, epoch_020.pt …` — periodic checkpoints
- `outputs/logs/train_log.csv` — per-epoch metrics
- `outputs/results/test_metrics.csv` — final test set evaluation
- `outputs/results/training_curves.png`

## Model Architecture

```
Input (224×224×3)
      ↓
ResNet-50 backbone (pretrained ImageNet)
      ↓ (B, 2048)
  ┌───────────────────────────┐
  │                           │
Intent Head              Deviation Head
Dropout(0.5)             Dropout(0.5)
Linear(2048→512)         Linear(2048→256)
ReLU                     ReLU
Linear(512→30)           Linear(256→4)
  │                           │
Softmax (30 classes)     Sigmoid (4 binary outputs)
```

**Loss:** `L = CrossEntropy(intent) + BCEWithLogitsLoss(deviation)`
Both losses equally weighted (λ1=λ2=1.0). `pos_weight` passed to BCE to handle 80/20 clean/deviated imbalance.

**Key classes:**
- `src/model.py` — `DrawNet`, `DrawNetLoss`
- `src/dataset.py` — `CachedDataset`, `build_dataloaders`
- `src/augment.py` — `generate_deviation_sample`

## Deviation Classes (4 total)

| Index | Name | What it simulates |
|---|---|---|
| 0 | `rotation` | Drawing rotated >15° from expected orientation |
| 1 | `closure_failure` | Gaps in shapes that should be closed |
| 2 | `spatial_disorganization` | Components shifted out of their expected positions |
| 3 | `size_distortion` | Drawing scaled 0.5–2× relative to canvas |

Deviations are applied synthetically via `augment.py` to 20% of the dataset. Within that 20%, each deviation fires independently at `p=0.6` with `min_deviations=1`.

## Design Decisions (context for future changes)

**Why 20% deviated rate?**
The original design applied deviations to ~85% of images (p=0.4 per class × 6 classes), making deviation detection trivially easy. 20% is more realistic and harder.

**Why stratified deviation flagging?**
`build_annotations.py` flags 20% per category (not 20% of the whole pool randomly). This ensures rare TU-Berlin categories (e.g. guitar with 255 images) still have deviated examples rather than all deviation flags falling on large categories.

**Why WeightedRandomSampler in training?**
QuickDraw categories have ~5,000 images each; TU-Berlin categories have 255–1,548. Without resampling, the model would rarely see TU-Berlin categories. `build_dataloaders` uses inverse-frequency weighting per intent class.

**Why separate build_annotations → cache → split instead of one script?**
- `build_annotations.py` is fast (<1 min) and lets you inspect label distribution before committing to the slow cache run.
- `cache_dataset.py` is the expensive step (~10–15 min). Supports resume if interrupted.
- `split_dataset.py` is deterministic and can be rerun with different seeds without re-caching.

**Why `pretrained: false` locally?**
SSL certificate issue on Windows blocks the PyTorch model download. Set `pretrained: true` on Colab/Linux.
